Nugget 08

Tensor parallelism: splitting weight matrices across GPUs

When a single weight matrix is too large (or too slow) for one GPU, it can be sliced into vertical columns and distributed across multiple GPUs. Each GPU computes its portion simultaneously; results are summed with an AllReduce. This requires very high-bandwidth interconnect because it happens inside every single layer.

The question

We've seen how tiling breaks a matmul into pieces within a single GPU. But what about when the model itself is too large for one GPU, or when we want more GPUs working on the same layer at the same time?

Mechanism

In tensor parallelism, the weight matrix W is split column-wise across N GPUs. Each GPU receives the full input activation A, multiplies it against its slice of W, and produces a partial output. These partial outputs are then summed across all GPUs via an AllReduce collective operation.

Input A (full, replicated on all GPUs)

GPU 0: A × W[:, 0:2048]   ──┐
GPU 1: A × W[:, 2048:4096] ──┼──► AllReduce (sum) ──► output
GPU 2: A × W[:, 4096:6144] ──┘

All GPUs participate simultaneously, then synchronize.

The AllReduce is the critical synchronization point. Every GPU must send its partial result to every other GPU, and all must wait for the sum before proceeding to the next layer. This is why tensor parallelism is so sensitive to interconnect speed.

Why NVLink, not PCIe

Tensor parallelism works within a single server node because NVLink provides ~900 GB/s of bidirectional bandwidth between GPUs — fast enough that the AllReduce cost is small relative to the compute time. Over PCIe (~64 GB/s), the communication would dominate and efficiency would collapse. Over InfiniBand (inter-node), it would be essentially unusable at this level of synchronization frequency.

InterconnectBandwidthTensor parallel viable?
NVLink (intra-node)~900 GB/sYes — designed for this
PCIe (intra-node)~64 GB/sMarginal — too slow
InfiniBand (inter-node)~50 GB/sNo — use pipeline parallelism instead

Why it matters

Tensor parallelism is what allows a single transformer layer to be computed by 8 GPUs simultaneously. Without it, very large layers would simply not fit in a single GPU's memory, and even if they did, one GPU would be the bottleneck while 7 others sat idle.

The tradeoff is tight coupling: all 8 GPUs must stay in lockstep, which means one slow GPU stalls all the others. This is why the 8 GPUs in a tensor-parallel group are always on the same physical server, connected by NVLink, with matched hardware.

How it combines with other strategies

In practice, tensor parallelism handles the within-node dimension of a much larger 3D parallelism scheme. Pipeline parallelism handles the between-node dimension (different nodes run different layers), and data parallelism handles the between-replica dimension (different node groups run different training examples). A 1,024-GPU training run might be: 8-way tensor parallel × 8-way pipeline parallel × 16-way data parallel.
