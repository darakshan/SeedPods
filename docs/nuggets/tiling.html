Nugget 04

Tiling: breaking big matmuls into pieces a GPU can hold

A GPU's arithmetic units are so fast that the bottleneck is almost always memory bandwidth. Tiling keeps data in fast on-chip memory as long as possible, maximizing the number of multiply-accumulates per byte loaded from slow main memory.

The question

GPUs can do enormous numbers of multiplications in parallel — but even so, a single large matrix multiply for a big transformer layer involves far more data than fits in the GPU's fastest memory. How does the hardware break it up?

Mechanism

A matrix multiply C = A × B where A is (M×K) and B is (K×N) requires M×N×K multiply-accumulate operations. For a large transformer layer, these dimensions can be 8192×8192 — way too large to fit in registers or shared memory (SRAM).

The solution is tiling (also called blocking): subdivide A and B into smaller rectangular tiles, compute partial results, and accumulate them into the output. Each output tile C[i,j] is the sum of a sequence of A-tile × B-tile products across the K dimension.

A (M×K)              B (K×N)
┌──┬──┬──┐           ┌──┬──┐
│A₀│A₁│A₂│     ×    │B₀│B₃│
├──┼──┼──┤           ├──┼──┤
│A₃│A₄│A₅│           │B₁│B₄│
└──┴──┴──┘           ├──┼──┤
                     │B₂│B₅│
                     └──┴──┘

C[0,0] = A₀×B₀ + A₁×B₁ + A₂×B₂   ← accumulates across K

Three levels of tiling

Tiling maps directly onto the GPU's three-level memory hierarchy, and each level requires a different tile size:

LevelUnitMemory usedTile size (typical)
GridStreaming Multiprocessors (SMs)HBM → SRAM128 × 128
Warp32-thread warps within an SMSRAM → registers64 × 64
ThreadIndividual threads / Tensor CoresRegisters only16 × 16

The K (inner) dimension is handled by iterating: each SM loops through K in chunks, accumulating partial sums into register-resident accumulators. This is what makes tiling different from just splitting in 2D — the reduction across K is sequential within each output tile.

Why it matters

The key metric is arithmetic intensity: how many floating-point operations you perform per byte loaded from slow memory. A perfectly tiled matrix multiply achieves intensity proportional to the tile size — meaning you load each byte once, then reuse it for an entire tile's worth of computation. This is why GPU utilization is so sensitive to matrix shape: small or "skinny" matrices tile poorly and leave the hardware waiting on memory.

FlashAttention applies the same insight to the attention operation, fusing the softmax and matmul into one kernel so intermediate N×N attention scores never have to be written back to HBM at all.

The punchline

Tiling is the bridge between the theoretical peak FLOP/s of a GPU and what you actually get in practice. All of the higher-level parallelism strategies (tensor parallelism, pipeline parallelism) are built on top of this foundation — they assume that within each GPU, tiling is already handling the memory hierarchy optimally.
